# F.R.I.D.A.Y. App Icon Specification

## Design Brief
Professional AI assistant app icon inspired by Iron Man HUD aesthetic.

## Color Palette
- **Background**: #0A0A0F (near black with blue tint)
- **Primary Accent**: #00D4FF (cyan - glowing, tech feel)
- **Secondary**: #0096FF (deeper blue)
- **Glow/Bloom**: #00D4FF with 40% opacity

## Icon Design Specifications

### Primary Icon (icon.png)
- **Size**: 1024x1024 px (for asset generation)
- **Design Elements**:
  - Dark background: #0A0A0F
  - Large glowing "F" in #00D4FF
  - Subtle circuit board pattern or concentric circles around the F
  - Soft glow/bloom effect using #00D4FF at 30-40% opacity
  - Sharp, minimal design - recognizable at 180px down to 180px size

### Adaptive Icon (adaptive-icon.png) - Android Only
- **Safe Zone**: 66dp circle in center (for adaptive masking)
- **Foreground**: Icon elements (same F design as primary)
- **Background**: Solid #0A0A0F or subtle gradient
- **Design**: Ensure F is centered and clear when masked to circles

### Splash Screen (splash.png)
- **Size**: 1280x2340 px (for scaling to all devices)
- **Design Elements**:
  - Background: Gradient from #0A0A0F to #12121A (subtle)
  - Centered F.R.I.D.A.Y. logo or large glowing F
  - "Loading Friday..." or "Initializing..." text centered below
  - Subtle animated scanlines or grid pattern (optional)
  - Keep loading time <1s

### Favicon (favicon.png)
- **Size**: 192x192 px
- **Design**: Simple glowing F on dark background
- **Usage**: Web version branding

## Icon Generation Instructions

### Using Figma (Recommended)
1. Create 1024x1024 artboard
2. Design glowing F with:
   - Bold sans-serif typeface
   - Outer glow filter: #00D4FF, blur 20px, spread 10px
   - Inner shadow: subtle depth
3. Export as PNG at 1x, 2x, 3x scales
4. Generate adaptive icon with safe zone guide

### Using CLI (with sharp/imagemagick)

```bash
# Install dependencies
npm install sharp

# Generate all icon sizes from 1024x1024 master
node scripts/generate-icons.js
```

See `scripts/generate-icons.js` for automated generation.

### Required Icon Assets (Post-Generation)
```
assets/
├── icon.png (1024x1024) - Primary app icon
├── adaptive-icon.png (1024x1024) - Android adaptive foreground
├── splash.png (1280x2340) - App launch splash screen
├── favicon.png (192x192) - Web favicon
└── (Generated by Expo at build time)
    ├── icon-180.png
    ├── icon-192.png
    ├── icon-512.png
    └── other platform-specific sizes
```

## Design Properties for Figma/Design Tool

### Typography
- Font: Inter, Helvetica Neue, or Montserrat (sans-serif)
- Weight: 700-900 (bold)
- Letter spacing: -0.02em (tight, modern)
- Size: Large (F should occupy 60-70% of canvas)

### Effects
- **Glow**: 
  - Color: #00D4FF
  - Blur: 15-20px
  - Spread: 5-8px
  - Opacity: 40%
  
- **Inner Depth**:
  - Shadow color: #000000
  - Blur: 10px
  - Offset Y: 2px
  - Opacity: 60%

- **Stroke** (Optional):
  - Color: #00D4FF
  - Width: 1-2px
  - Opacity: 60%

## Testing Checklist
- [ ] Icon readable at 40x40 px (smallest size)
- [ ] Icon recognizable at 180px (app icon size)
- [ ] Glow effect visible but not overwhelming
- [ ] Colors match theme (accent #00D4FF)
- [ ] Splash screen loads quickly (<1s)
- [ ] All required sizes generated
- [ ] No transparency issues on different backgrounds

## App Store Guidelines
- **iOS**: Icon must be JPEG/PNG, 1024x1024, RGB color space
- **Android**: Follow adaptive icon guidelines (safe zone 72px from edge)
- **Web**: Favicon must be exactly 192x192 PNG

## Version Control
- Store 1024x1024 master icons in version control
- Generated sizes (Expo build artifacts) should NOT be in version control
- Use `.gitignore` to exclude `ios/` and `android/` build outputs

## Next Steps
1. Design icon in Figma/Adobe XD following specifications above
2. Export as PNG (1024x1024)
3. Place in `assets/` directory
4. Run `npm run build` (Expo will generate platform-specific sizes)
5. Test in iOS Simulator: `npm run ios`
6. Test in Android Emulator: `npm run android`
